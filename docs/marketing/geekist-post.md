# 지피터스 포스팅 초안

## 제목
주말에 만든 "서울 얼리버드 카페 지도" — 3000개 카페 데이터로 아침을 바꿨습니다

## 본문

### 만들게 된 계기

아침 일찍 카페에서 하루를 시작하고 싶은데, 매번 "강남 새벽 카페", "성수동 아침 카페"를 검색하는 게 너무 비효율적이었습니다. 카카오맵으로 검색해도 영업시간 필터가 없어서, 지도에 뜨는 카페 하나하나 열어보며 "이 카페 7시에 여나?" 확인해야 했습니다.

"아침 일찍 여는 카페만 모아서 지도로 보여주면 어떨까?"

### 만든 것

**모닝카페** — 서울에서 아침 일찍(6~8시) 여는 카페를 카카오 지도에서 한눈에 찾아주는 웹앱

🔗 **https://morning-cafe-phi.vercel.app**

### 핵심 기능

- **시간 필터**: ~6시, 6~7시, 7~8시 대역으로 원하는 오픈 시간대만 필터
- **요일별 영업시간**: 주말과 평일 오픈 시간이 다른 카페도 정확히 표시
- **구 필터**: 강남구, 마포구 등 25개 구별로 빠르게 좁히기
- **체인/개인 필터**: 스타벅스 빼고 개인 카페만 보기
- **바텀시트 상세**: 사진, 메뉴, 별점, 주차, 편의시설 한눈에
- **카카오톡 공유**: 발견한 카페를 친구에게 바로 공유
- **PWA**: 홈 화면에 추가하면 네이티브 앱처럼 사용

### 기술 스택

| 레이어 | 기술 |
|--------|------|
| 프론트엔드 | Next.js 16 (App Router) + TypeScript |
| 스타일링 | Tailwind CSS 4 |

### 데이터

- **9,400+** 서울 카페 데이터 (Supabase)
- **3,000+** 얼리버드 카페 (8시 이전 오픈)
- **830+** 24시간 카페
- **25개 구** 전체 커버

### 어려웠던 점

1. **네이버 이미지 프록시 문제**: 카카오맵 사진 중 Naver pstatic 호스팅 이미지가 Referer 제한으로 로딩 실패 → kakaocdn cthumb 프록시로 해결
2. **요일별 영업시간 파싱**: `hours_by_day` JSON에서 해당 요일 키가 없으면 "정보 없음"으로 처리해야 하는데, 기본 `opening_time`으로 fallback하는 버그가 있었음 → 주말에 9시 오픈 카페가 평일 7시 필터에 잡히던 문제 수정
3. **3000+ 마커 렌더링 성능**: 전체 마커를 한번에 그리면 메모리 폭발 → 뷰포트 기반 필터링으로 화면에 보이는 50~200개만 렌더링
