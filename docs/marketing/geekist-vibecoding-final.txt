===========================
소개
===========================

아침 러닝 끝나고 분위기 있는 카페 가고 싶을 때 있잖아요.
네이버 지도에서 시간 필터 걸면 — 스타벅스, 메가커피, 이디야만 나옵니다.

"프랜차이즈 빼고 아침에 여는 개인 카페만 보고 싶은데?"

그래서 만들었습니다.
"모닝커피" — 서울 아침 카페 지도.

👉 https://morning-cafe-phi.vercel.app

• 3,000개 얼리버드 카페
• 6시 / 7시 / 8시 시간대별 필터
• 프랜차이즈 on/off (이게 핵심)
• 25개 구별 필터
• 요일별 영업시간 (주말 다른 카페도 정확히)
• 바텀시트 상세: 사진, 메뉴, 별점, 주차, 편의시설
• 카카오톡 공유: 발견한 카페를 친구에게 바로 공유
• PWA: 홈 화면에 추가하면 앱처럼 사용

===========================
진행 방법
===========================

사용한 도구:
• Claude Code (CLI) — 전체 코딩, 아키텍처, 디버깅 전부
• Next.js 16 (App Router) + TypeScript
• Supabase (PostgreSQL) — 9,400개 카페 데이터
• 카카오맵 SDK — 지도 + 마커
• Tailwind CSS 4 + Zustand 5
• Vercel 배포, Serwist PWA

전체 개발을 Claude Code CLI로 했습니다. 프롬프트 → 코드 생성 → 빌드 확인 → 반복.


1단계: 프로젝트 셋업

「서울 카페 데이터를 Supabase에 저장하고, 카카오 지도 위에 마커로 표시하는 웹앱을 만들어줘. 핵심: 시간대별 필터, 구별 필터, 체인/개인 필터. 모바일 우선.」

→ 프로젝트 구조, DB 스키마, 스토어 설계까지 한번에 잡아줌.


2단계: 성능

「3000개 마커 한번에 렌더링하면 느려. 뷰포트 기반으로 화면에 보이는 것만 렌더링해줘. 250ms 쓰로틀.」

→ 뷰포트 필터링 + 쓰로틀 + SVG 마커 캐싱. 3,000개도 버벅임 없음.


3단계: 공유 기능

「카페 개별 페이지 /cafe/[id] 만들어줘. OG 이미지 동적 생성, 카카오톡 공유 Feed 템플릿.」

→ 카카오톡 공유하면 카페명 + 오픈시간 + 주소 카드가 나옴.

[스크린샷: 카카오톡 공유 카드]


핵심 팁: CLAUDE.md

프로젝트 루트에 CLAUDE.md를 상세하게 쓰면 Claude Code가 맥락을 정확히 이해합니다. 저는 여기에 프로젝트 구조, 코딩 컨벤션 19개, API 스펙, 레이아웃 규칙을 전부 기록해뒀습니다. 바이브코딩의 핵심은 CLAUDE.md입니다.

[스크린샷: 카페 상세 페이지 or 리스트 뷰]


===========================
결과와 배운 점
===========================

배운 점:

1. CLAUDE.md가 80%다
프롬프트보다 프로젝트 컨텍스트가 중요합니다. CLAUDE.md에 아키텍처, 컨벤션을 정리해두면 Claude Code가 일관된 코드를 생성합니다.

2. 작게 쪼개서 요청
"필터 만들어줘"가 아니라 "시간 필터 칩 UI" → "필터 변경 시 마커 업데이트" 이렇게 쪼개고, 매번 빌드로 확인.

3. 에러는 그대로 붙여넣기
빌드 에러 나면 그대로 Claude Code에 붙여넣으면 됩니다. TypeScript 타입 에러는 사람보다 빠름.


시행착오:

1. 3,000개 마커 메모리 폭발
전체 마커 한번에 렌더링 → 모바일 멈춤. 뷰포트 기반 필터링으로 해결.

2. 요일별 영업시간 버그
토요일 9시 카페가 평일 7시 필터에 잡히는 버그. hours_by_day에서 해당 요일 키 없으면 opening_time으로 fallback하던 게 원인. "주말 필터 이상해"라고 했더니 바로 잡아줌.

3. 카카오톡 공유가 텍스트만 나옴
OG 이미지 파일 누락 + 카카오 SDK 미초기화. 증상 그대로 전달하니 둘 다 한번에 해결.


앞으로의 계획:
• 카페 사장님 직접 등록 기능
• 데이터 자동 갱신 크롤링
• 인스타그램 구별 아침 카페 카드뉴스 연재
